# 自动宏观判定与资产配置系统

## 系统概述

本系统是一套基于\*\*“普林格六阶段”理论**构建的量化决策工具，旨在通过**8 个核心宏观指标**的自动化处理，实现从“数据输入”到“资产配置比例输出”的全流程闭环。系统严格遵循**“四维六指标”判定体系\*\*，并内置\*\*“红利≤30%/商品≤20%”双封顶约束**及**情绪过热/冰点保护机制\*\*，帮助投资者在复杂宏观环境中保持纪律性配置，规避主观情绪干扰。

## 核心功能

- **宏观周期判定**：基于经济增长、通货膨胀、资金流动性、市场情绪四个维度，自动判定当前所处的宏观经济阶段
- **资产配置建议**：根据判定的宏观阶段，自动输出各类资产的配置比例，包含成长股、红利股、商品ETF、债券/现金
- **ETF标的智能配置**：基于多因子评分模型，为15只核心ETF标的分配权重
- **数据自动获取**：通过GitHub Actions定期获取最新宏观数据，确保数据的及时性
- **设置管理**：支持对系统基础数据的编辑和管理，包括配置矩阵、阶段描述、ETF标的池等
- **历史记录**：保存最近12条配置记录，便于复盘和分析

## 技术实现

- **前端技术**：HTML5 + CSS3 + JavaScript
- **样式框架**：Tailwind CSS
- **数据可视化**：Chart.js
- **数据存储**：localStorage
- **数据获取**：Python + AkShare + GitHub Actions
- **部署方式**：静态网页，支持GitHub Pages、Gitee Pages等静态托管服务

## 系统架构

1. **数据输入层**：用户输入8个核心宏观指标和ETF标的数据
2. **逻辑引擎层**：计算四维得分，判定宏观阶段
3. **配置计算层**：根据宏观阶段计算资产配置比例，应用双封顶约束和情绪保护机制
4. **ETF配置层**：基于多因子评分模型，为ETF标的分配权重
5. **结果展示层**：展示配置结果、ETF详细配置、多因子评分结果和历史记录

## 部署步骤

### 1. 克隆或下载项目

### 2. 部署到静态托管服务

#### GitHub Pages

1. 在GitHub上创建新仓库
2. 上传项目文件（包括`.github`目录）
3. 进入仓库设置，启用GitHub Pages
4. 手动触发一次数据获取 workflow
5. 访问生成的URL

#### Gitee Pages

1. 在Gitee上创建新仓库
2. 上传项目文件（包括`.github`目录）
3. 进入仓库设置，启用Gitee Pages
4. 手动触发一次数据获取 workflow
5. 访问生成的URL

### 3. 配置GitHub Actions

项目已包含`.github/workflows/fetch-data.yml`文件，该文件会：

- 每天自动运行数据获取脚本
- 使用AkShare获取最新宏观数据
- 将数据保存为`data.json`文件
- 自动提交更新到仓库

### 4. 首次使用

1. 访问系统URL
2. 点击"自动获取数据"按钮，系统会从`data.json`文件加载最新数据
3. 点击"计算配置"按钮，获取资产配置建议
4. 如需修改系统设置，点击"系统设置"链接，使用默认密码`admin123`登录

## 数据说明

### 核心宏观指标

| 维度         | 核心指标            | 数据源     | 判定逻辑与阈值                                                                                 |
| :--------- | :-------------- | :------ | :-------------------------------------------------------------------------------------- |
| **经济增长**   | **制造业 PMI**     | AkShare | **>50**：扩张 (得分 +1)**<50**：收缩 (得分 -1)                                                    |
| <br />     | **社融存量同比增速**    | AkShare | **回升**：宽信用 (领先经济 1-2 季度)**下降**：紧信用                                                      |
| **通货膨胀**   | **CPI 同比**      | AkShare | **快速上升**：过热/滞胀风险**低位/负值**：衰退/复苏初期                                                       |
| <br />     | **PPI 同比**      | AkShare | **上行周期**：商品牛市 (利好商品 ETF)**下行周期**：工业通缩                                                   |
| **资金/流动性** | **M1-M2 剪刀差**   | AkShare | **走阔 (M1>M2)**：资金活化 (股市活跃/过热)**收窄/负值**：资金定期化 (避险)                                       |
| <br />     | **10 年期国债收益率**  | AkShare | **下行**：利好债券/成长股**快速上行**：利好价值/商品，利空高估值                                                   |
| **市场情绪**   | **中证全指成交额滚动环比动量** | AkShare | `turnoverMomentum = 0.7 × 5日滚动环比 + 0.3 × 20日滚动环比`；**>0 且继续增强**：情绪复苏；**>50%**：情绪过热；**< -20%**：情绪冰点 |
| <br />     | **股债利差 (ERP)**  | 计算得出    | **>80% 分位**：股票极便宜 (左侧布局)**<20% 分位**：股票极贵 (右侧减仓)                                         |

### ETF标的池

- **成长股ETF**：宽基成长、科技成长、跨境成长
- **红利股ETF**：红利ETF、红利低波ETF
- **商品ETF**：黄金ETF、原油基金、有色金属ETF
- **债券ETF**：10年国债ETF
- **现金ETF**：银华日利

## 风险提示

1. **数据滞后性**：宏观指标存在发布滞后，系统判定反映的是“过去一个月”的状态
2. **阈值适应性**：部分阈值基于历史数据设定，若市场环境发生变化，需手动调整
3. **黑天鹅事件**：系统基于历史规律构建，对突发性事件反应可能滞后
4. **非上市公司数据**：本系统仅适用于公开市场资产

## 系统维护

1. **数据更新**：GitHub Actions会每天自动更新数据，确保数据的及时性
2. **系统设置**：通过设置页面可以修改配置矩阵、阶段描述、ETF标的池等基础数据
3. **密码管理**：首次登录设置页面使用默认密码`admin123`，登录后建议修改密码
